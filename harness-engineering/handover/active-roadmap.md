# 활성 로드맵 (Active Roadmap)

> **이 파일은 세션 시작 시 반드시 읽는다.**
> 세션이 바뀌어도 "지금 뭘 하고 있고, 다음에 뭘 해야 하는지"를 잃지 않기 위한 파일.
> 작업 완료 시마다 이 파일을 업데이트한다.

---

## 현재 대목표: FMI 재무 자동화 파이프라인

### 전체 비전

운영 중인 차량에 계약된 고객(투자자)에게 차량의 수입/지출을 보여주고,
계약 내용대로 금액을 정산해주는 과정에서 통장/카드 사용내역이 **증빙자료**가 되고 **자동화**되어 운영.

```
PHASE 1 (수집) → PHASE 2 (1차 분류) → PHASE 3 (2·3차 분류) → PHASE 4 (손익+정산) → PHASE 5 (투명성)
```

---

## PHASE 1 — 수집

| 항목 | 상태 | 설명 |
|------|------|------|
| SMS 수신 (SMS Forwarder → 웹훅) | ✅ 구현됨 | POST /api/finance/sms-webhook |
| 파싱 + 분류 (카드승인 / 은행입출금 / 일반문자→ignored) | ✅ 구현됨 | sms-parsers.ts v3 |
| 엑셀 업로드 (검증용) | ✅ 기존 존재 | 수동 검증/보정용으로 유지 |

### 지원 카드사/은행

| 발급사 | 타입 | 파서 상태 |
|--------|------|----------|
| KB국민카드 | 카드 | ✅ 3가지 포맷 |
| 우리카드 | 카드 | ✅ 3가지 포맷 |
| 현대카드 | 카드 | ✅ 2가지 포맷 |
| MY COMPANY | 법인카드 | ✅ 2가지 포맷 |
| 우리은행 | 은행 | ✅ 출금/입금 |
| KB은행(국민은행) | 은행 | ⚠️ 기본 준비 (실제 데이터 도착 시 보정) |

---

## PHASE 2 — 1차 자동분류 (카드/통장 → 차량·업무)

> 카드와 통장을 차량 또는 업무에 매핑하는 단계

| 항목 | 상태 | 설명 |
|------|------|------|
| **카드별 매핑** | 🔲 설계 필요 | 카드 A → 12가3456 전용, 카드 B → 공용(2차 분류로) |
| **통장별 매핑** | 🔲 설계 필요 | A통장 → 렌트수입, B통장 → 운영비 |
| **자동 분류 완료** | 🔲 설계 필요 | 전용카드 → 즉시 확정, 전용통장 → 즉시 확정 |

### 핵심 설계 포인트
- 카드/통장 → 차량 매핑 테이블 필요 (card_alias → car_id or '공용')
- 전용 카드/통장은 1차에서 바로 차량에 확정 배정
- 공용 카드/통장은 PHASE 3으로 넘어감

---

## PHASE 3 — 2·3차 분류 (공용카드·통장 세부 분류)

> 공용 카드/통장의 거래를 세부 분류하는 단계

| 항목 | 상태 | 설명 |
|------|------|------|
| **규칙 기반 분류** | 🔲 설계 필요 | 가맹점명 → 주유/정비/보험, 적요에 차량번호 포함 시 매칭 |
| **AI 매칭** | 🔲 설계 필요 | Gemini 기반 유사 패턴, 과거 분류 학습 |
| **수동 확인** | 🔲 설계 필요 | 자동 분류 불가 건 → 사용자 최종 확정 |

### 핵심 설계 포인트
- 분류 카테고리: 주유비, 정비비, 보험료, 렌트수입, 운영비, 기타
- AI는 과거 분류 이력을 학습해서 점점 자동화율 올림
- 수동 확인 건이 줄어드는 것이 목표

---

## PHASE 4 — 차량별 손익 + 정산

| 항목 | 상태 | 설명 |
|------|------|------|
| **차량별 손익** | 🔲 설계 필요 | 수입(렌트료) - 지출(정비·유류·보험) = 월별 순이익 |
| **투자 정산** | 🔲 설계 필요 | 계약 조건 × 차량손익 = 투자자별 정산액 |
| **경영 관리** | 🔲 설계 필요 | 급여/운영비/회계, 전체 손익 리포트 |

---

## PHASE 5 — 투명성

| 항목 | 상태 | 설명 |
|------|------|------|
| **투자자 리포트** | 🔲 최종목표 | 증빙 첨부 → 기존 시스템 데이터와 크로스체크 → 신뢰 확보 후 전환 |

---

## 사용자 결정 사항

- 오픈뱅킹 → 비용 이슈로 보류, SMS로 진행
- 엑셀 업로드 → 검증/보정용으로 유지
- SMS Forwarder → 모든 문자 수신 (필터 없음, 시스템에서 분류)
- 은행 SMS도 같은 앱으로 같은 폰에서 수신 중
- 불가 건 외에는 전부 자동화
