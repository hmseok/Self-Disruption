## 현재 대목표: 재무 자동화 파이프라인

### 전체 비전 (사용자 기획)

```
SMS 자동수집 (카드+은행)
    ↓
자동 분류 (카드사/은행별)
    ↓
차량별 수입/지출 매핑 (Vehicle P&L)
    ↓
투자자 정산 (계약 기반)
    ↓
투명한 운영 보고서
```

**핵심 원칙:**
- 운영 중인 차량에 계약된 고객(투자자)에게 차량의 수입/지출을 보여주고
- 계약 내용대로 금액을 정산해주는 과정에서
- 통장/카드 사용내역이 **증빙자료**가 되고 **자동화**되어 운영
- 오픈뱅킹은 비용 이슈로 보류 → SMS로 진행
- 엑셀 업로드는 검증용으로 유지

### 파이프라인 단계별 상태

| # | 단계 | 상태 | 설명 |
|---|------|------|------|
| 1 | **SMS 자동수집** | ✅ 완료 | SMS Forwarder → webhook → DB 저장 |
| 2 | **카드 SMS 파싱** | ✅ 완료 | KB, 우리, 현대, MY COMPANY 파서 v3 |
| 3 | **은행 SMS 파싱** | ✅ 완료 | 우리은행 출금/입금, KB은행 기본 준비 |
| 4 | **비금융 SMS 무시** | ✅ 완료 | 'ignored' 상태로 자동 분류 |
| 5 | **실패 건 재파싱** | ✅ 완료 | POST /api/finance/sms로 재처리 |
| 6 | **SMS → 거래 테이블 연동** | 🔲 미착수 | card_sms_transactions → transactions 자동 적재 |
| 7 | **카드/통장 → 차량 매핑** | 🔲 미착수 | 1차: 가맹점 규칙, 2차: AI 분류 |
| 8 | **차량별 P&L (수익/비용)** | 🔲 미착수 | 차량별 수입(렌트료) vs 지출(유류/정비/보험 등) |
| 9 | **투자자 정산** | 🔲 미착수 | 계약 조건 기반 정산 계산 + 보고서 |
| 10 | **운영 보고서/대시보드** | 🔲 미착수 | 투자자 포털, 차량별 투명 보고 |

## 직전 완료 작업

- SMS 파서 v3: 카드 + 은행 통합 (2026-04-25)
- 실패 건 재파싱 → 10건 중 6건 parsed, 4건 ignored, 0건 failed

## 다음 작업 (우선순위 순)

1. **SMS → transactions 자동 적재** (단계 6)
   - parsed된 SMS를 transactions 테이블로 자동 이관
   - 중복 방지 (sms_id 참조)
   - 카드 결제 → expense, 은행 입금 → income, 은행 출금 → expense

2. **카드/통장 → 차량 매핑 규칙** (단계 7)
   - 가맹점명 기반 자동 매핑 (주유소→유류비, 정비소→정비비)
   - 차량 배정 규칙 (카드별 → 차량 고정 배정 or 수동 매핑)

3. **차량별 P&L** (단계 8)
   - 차량별 수입: 렌트 계약 월 청구액
   - 차량별 지출: 매핑된 카드/통장 거래
   - 월별 손익 집계

4. **투자자 정산** (단계 9)
   - 계약서 기반 정산 비율/금액 계산
   - 정산 보고서 생성
   - 투자자 열람 포털

## 사용자 선호/결정 사항

- 오픈뱅킹 → 비용 이슈로 보류, SMS로 진행
- 엑셀 업로드 → 검증/보정용으로 유지
- SMS Forwarder → 모든 문자 수신 (필터 없음, 시스템에서 분류)
- 은행 SMS도 같은 앱으로 같은 폰에서 수신 중
- 불가 건 외에는 전부 자동화
