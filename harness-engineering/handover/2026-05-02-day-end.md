코드 작성보다 **검증 도구 먼저**:

### 1. alias 예약어 검사 (sql-reserved-alias-lint.js)
- 검사: `AS (desc|order|group|select|from|where|key|index|...)` 패턴
- 위반 시 commit 차단
- MySQL 예약어 목록은 공식 문서 참고

### 2. GROUP BY ↔ SELECT expression 정합성 (sql-group-by-lint.js)
- 검사: `GROUP BY <alias>` 패턴 발견 시 SELECT 의 같은 alias 가 단순 컬럼인지 expression 인지 확인
- expression 이면 GROUP BY 도 expression 으로 명시 강제
- only_full_group_by 모드 호환

### 3. JSON 함수 MySQL 버전 호환성
- 기존 sql-fn-lint.js 확장
- JSON_REMOVE, JSON_TABLE, JSON_SCHEMA_VALID 등 5.7 vs 8.0+ 분류
- MySQL 5.7 환경 가정 — 8.0+ 전용 함수 사용 시 명시 주석 필요

### 4. alias-prefixed 컬럼 검증 강화 (sql-lint.js 확장)
- 현재: 단순 컬럼명 → 모든 테이블 검색 → ambiguous 시 false negative
- 개선: `<alias>.<column>` 패턴이면 alias → 테이블 매핑 후 정확히 검증
- FROM/JOIN 절에서 alias-table 매핑 추출
