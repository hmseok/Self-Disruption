# 회귀 케이스 — 분류 검수 화면 취소/승인 구분 누락

**날짜**: 2026-05-01
**카테고리**: ui-data-coverage, review-screen
**심각도**: 🔴 Critical (사용자 검수 불가능 → ERP 가치 상실)

---

## Input

분류 검수 → 「유류비」 카테고리 클릭 → 거래 목록

## Expected (사용자가 보고 싶은 화면)

```
날짜       유형      거래처    적요               금액           소스
04-30      [지출]    석호민    (주)잠실에너지      91,000        SMS
04-30      [지출]    석호민    (주)잠실에너지      180,000       SMS
04-30      [취소]    석호민    [취소] (주)잠실에너지 -180,000     SMS  ← 빨강 강조
```

## Actual (수정 전)

```
날짜       유형      거래처    적요              금액         소스
04-30      [지출]    석호민    (주)잠실에너지     91,000      SMS
04-30      [지출]    석호민    (주)잠실에너지     180,000     SMS
04-30      [지출]    석호민    (주)잠실에너지     180,000     SMS  ← 취소인데 지출로 표시
```

→ 사용자: "취소,승인의 기준도 뭐지 지출이라고 되어있고 제대로 하고있는것 맞아요?"

---

## Root Cause — 3-Why

| Why | 답 |
|-----|----|
| 1. 왜 취소 거래가 지출로 표시되나 | UI 가 transactions.type 만 보고 sms_transaction_type 무시 |
| 2. 왜 sms_transaction_type 활용 안 했나 | 카드 거래 탭만 수정하고 분류 검수 화면은 같이 수정 안 함 |
| 3. 왜 한 화면만 수정했나 | 같은 데이터 필드를 표출하는 모든 화면을 인덱스화 안 함 |

## Prevention — 재발 방지

CLAUDE.md § 0-1 **규칙 12 신설** (2026-05-01):

```
UI 화면별 데이터 정합성 자가 검증
[A] UI 화면 인덱스 확보 — grep 으로 전수 조사
[B] 검수 시나리오 명시 — 사용자 관점
[C] 데이터 중복/오염 자동 감지
[D] 사용자 검수 가능성 보장 — 색상/라벨로 즉시 식별
```

- ⏳ 정산 연결 화면 (검사 필요)

## Test Vectors

분류 검수 화면 거래 목록에서:
- canceled SMS row → 빨강 [취소] 라벨 + 음수 금액 + 연빨강 배경

- deposit → 파랑 +금액
- withdrawal → 빨강 -금액
- (sms_transaction_type 없음) → type 기반 fallback
