## 2. 🔴 치명적 이슈 (즉시 수정 필요)

### 2.1 직원초대 비밀번호 미저장 (Critical)
- **파일**: `app/api/member-invite/accept/route.ts`
- **문제**: `password`를 받지만 **해싱도 저장도 하지 않음**
- **영향**: 초대 수락 → 프로필 생성 → 비밀번호 없음 → **로그인 불가**
- **해결**: bcryptjs 해싱 + `password_hash` 컬럼에 INSERT/UPDATE 추가

### 2.2 SQL Injection 취약점 (9개 라우트)
- `app/api/cars/[id]/route.ts` — PATCH에서 문자열 보간
- `app/api/accident-records/[id]/route.ts` — Object.keys 직접 보간
- `app/api/e-contract/[id]/route.ts` — 문자열 보간 UPDATE
- `app/api/e-contract/route.ts` — INSERT에서 동적 SQL
- `app/api/finance/import-excel/route.ts` — IN 절 문자열 조합
- `app/api/admin/contracts/route.ts` — LIKE 파라미터 미이스케이프
- `app/api/car-costs/route.ts` — 동적 쿼리 연결
- `app/api/assignments/route.ts` — 인증 없음 + 동적 SQL
- `app/api/temp-query/route.ts` — 디버그 엔드포인트 다수 취약점

### 2.3 DB 테이블 미존재 가능성
- `member_invitations` — 직원초대 핵심 테이블
- `user_page_permissions` — 페이지 권한 테이블
- `quotes`, `quote_share_tokens` — 견적 테이블
- `quote_lifecycle_events` — 견적 이력 테이블
- Prisma 마이그레이션 히스토리 없음 → 수동 테이블 관리

## 3. ⚠️ 높은 우선순위 이슈

### 3.1 Prisma 스키마 불일치 (45.7% 커버리지)
- 42개 Prisma 모델 vs 92개 실제 테이블
- 50+ 테이블이 Raw SQL로만 접근 (타입 안전성 없음)
- 누락된 핵심 모델: employees, positions, departments, customers, contract_terms 등

### 3.2 단독회사 전환 미완료
- 13+ 모델에 아직 `company_id` 필드 존재
- API 호환성을 위해 `verifyUser`에서 companies 테이블 조회 유지 중
- 일부 API에서 `company_id` 필터링이 데이터 누락 유발 가능

### 3.3 인증 불일치
- 일부 라우트: `verifyUser` 사용
- 일부 라우트: 커스텀 JWT 디코딩 사용
- 일부 라우트: 인증 없음 (assignments GET 등)

## 6. 오늘 4개 작업 실행 가능성 평가

### 작업 1: 직원초대 → 페이지권한 → 견적 접근
- **코드 완성도**: 90%
- **필요 작업**:
  1. accept에 bcrypt 해싱 + password_hash 저장 (30분)
  2. DB 테이블 존재 확인/생성 (30분)
  3. 페이지 권한에 `/quotes/pricing` 포함 확인 (15분)
- **예상 소요**: 1~1.5시간
- **위험도**: 낮음 — 코드 거의 완성, DB 테이블만 확인

### 작업 2: 견적 고객 전달
- **코드 완성도**: 95%
- **필요 작업**:
  1. DB 테이블 존재 확인 (quotes, quote_share_tokens)
  2. 공유 링크 생성 → 고객 접근 → 서명 플로우 검증
- **예상 소요**: 30분~1시간
- **위험도**: 낮음

### 작업 3: 통장/카드 API (Codef)
- **코드 완성도**: 85%
- **필요 작업**:
  1. Codef API 키 확인 (환경변수)
  2. 은행/카드 계정 연결 테스트
  3. 거래내역 동기화 검증
- **예상 소요**: 2~3시간
- **위험도**: 중간 — Codef 응답 없으면 대안 탐색 필요

### 작업 4: 투자자 지급금액
- **코드 완성도**: 70%
- **필요 작업**:
  1. `/finance/settlement` 페이지 현재 기능 확인
  2. 투자자별 정산 계산 로직 검증
  3. 지급금액 확정 UI/API 추가
- **예상 소요**: 2~3시간
- **위험도**: 중간
