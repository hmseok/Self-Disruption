## 8. 사용자 추가 지시사항 (2026-04-18)

> **"외부 실시간 조회 + 자체 운영데이터 = 기본 DB → 운영하면서 일부씩 조정 가능"**

### 해석 및 적용 원칙
- **초기 기본 DB 구축**: ①외부 크롤링 실시간 시세 + ②자체 보험/통장/원가 실데이터 → 두 데이터 소스를 결합하여 기본 DB 시드
- **점진 조정 메커니즘**: 운영학습 엔진(Phase 2)이 수집하는 실적(vehicle_operations / calc_snapshots / operational_actuals)을 기반으로 필요 시 부분 조정
- **불변/가변 구분 필요**: Planner 단계에서 "법정값(고정)" vs "시장값(가변)" 구분 설계

### Planner가 설계해야 할 3가지 데이터 통합 룰
1. **외부 시세 → vehicle_market_price 테이블**: 자체 9대 + 추후 운영 차종
   - 소스: 엔카/KB차차차/중고차 시세 사이트 (우선 1~2곳) — WebFetch 기반 정기 갱신
   - 주기: 월 1회 (Scheduled Task 사용 고려)
2. **자체 운영비 → business_rules 덮어쓰기**: OVERHEAD, MAINTENANCE, LOAN_INTEREST는 자체 실적이 우선
3. **점진 조정 인터페이스**: 기존 기준표 UI에 "현재값 / 외부추천값 / 운영실적" 3열 병행 표시 → 관리자가 선택 가능
