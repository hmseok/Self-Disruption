## 1. 페르소나

### 페르소나 1 — 권한자 (총무팀)

| 항목 | 내용 |
|------|------|
| 식별 | `ride_asset_admins.user_id` 화이트리스트 (라이드 admin 이 등록) |
| 대표 사용자 | 총무팀원 1~3명 |
| 주 업무 | 자산 등록 → QR 라벨 인쇄 → 부착 → 사용자 매칭 → 회수/처분 처리 |
| 사용 디바이스 | 데스크톱 (등록·인쇄) + 모바일 (스캔 확인) |
| 접근 범위 | **전체 자산** (모든 카테고리, 모든 사용자 할당분) |

### 페르소나 2 — 일반 사용자 (자산 할당받은 직원)

| 항목 | 내용 |
|------|------|
| 식별 | `ride_assets.assigned_user_id = 본인` 인 자산이 1건 이상 |
| 대표 사용자 | 차량/노트북/법인카드 등 보유 직원 |
| 주 업무 | 본인 매칭 자산 목록 조회, 위치/메모 업데이트 |
| 사용 디바이스 | 모바일 우선 (QR 스캔), 데스크톱 보조 |
| 접근 범위 | **본인 매칭 자산만** (다른 사용자/공통 자산 불가) |

### 페르소나 3 — 라이드 admin (시스템 관리자)

| 항목 | 내용 |
|------|------|
| 식별 | `users.role = 'admin'` |
| 주 업무 | `ride_asset_admins` 화이트리스트 관리 (권한자 추가/제거) |
| 권한 | 권한자 동일 + 권한자 등록 |

## 2. 시나리오

### 시나리오 A — 총무팀, 신규 IT장비 (노트북) 입고 → 직원에게 매칭

> 상황: 총무팀이 노트북 5대 새로 받음. 5대 각각 등록 → QR 라벨 인쇄 → 부착 → 5명 직원 매칭.

1. **Step 1**: `/RideAssets` 진입 → 헤더 「📋 전체 자산」 탭 자동.
2. **Step 2**: DcStatStrip 상단 액션 버튼 「+ 자산 등록」 클릭 → 등록 모달.
3. **Step 3**: 모달에서 카테고리 「IT장비」, 자산명 「ThinkPad X1 Carbon Gen11」, 취득가/취득일 입력 → 5회 반복 (또는 일괄 등록 기능 — Phase 2).
4. **Step 4**: 자산코드 자동 부여 (`IT-2026-0001` ~ `IT-2026-0005`). 등록 즉시 `qr_token` UUID 생성.
5. **Step 5**: NavTabs 「🖨️ QR 인쇄」 클릭 → 방금 등록한 5개 다중 선택 → 「라벨지 PDF 출력」 클릭.
6. **Step 6**: PDF 다운로드 → A4 라벨지 인쇄 → 노트북 5대에 스티커 부착.
7. **Step 7**: 「📋 전체 자산」 복귀 → 각 자산 행 클릭 → 편집 모달 → `assigned_user_id` 직원 선택 → 저장.
8. **결과**: 5대가 5명에게 매칭됨. 각 직원이 본인 모바일에서 QR 스캔하면 자산 정보 확인 가능.

### 시나리오 B — 일반 사용자, QR 스캔으로 자산 정보 확인

> 상황: 직원이 책상에 부착된 QR 스티커 폰으로 스캔.

1. **Step 1**: QR 스캔 → 브라우저 자동 오픈 → `/RideAssets/qr/<qr_token>` (모바일 전용 라우트).
2. **Step 2**: 로그인 안 된 상태면 → 로그인 페이지 → 복귀.
3. **Step 3**: 모바일 페이지 표시:
   - 자산명, 카테고리, 자산코드
   - 매칭 사용자 (본인이면 「본인 자산」 강조)
   - 상태, 위치, 메모
   - **본인 자산이면** → 「위치 업데이트」 / 「메모 추가」 버튼 활성
   - **본인 자산 아니면** → 읽기 전용 (총무팀 외엔 위치만 표시, 사용자명 마스킹)
4. **Step 4**: 본인 자산 → 위치 변경 「3F 회의실 → 본인 자리」 → 저장 → `ride_asset_logs` 기록.

### 시나리오 C — 일반 사용자, 본인 자산 목록 조회 (데스크톱)

1. `/RideAssets` 진입.
2. NavTabs 「👤 내 자산」 자동 활성 (일반 사용자 진입 시 기본).
3. 본인에게 매칭된 자산만 카드/테이블 표시.
4. 각 행 클릭 → 상세 모달 (위치/메모 수정 가능, 카테고리/자산코드 등은 read-only).

### 시나리오 D — 총무팀, 자산 회수/처분

1. 직원 퇴사 등 → 「📋 전체 자산」 → 해당 자산 검색.
2. 편집 모달 → `assigned_user_id = null` 로 변경 → 상태 「공통 자산」으로 변경 → 저장.
3. 또는 상태 「처분(매각/폐기)」 선택 → `disposed_at` 자동 기록 → 목록에서 분리.
4. 「📦 공통 자산」 탭에서 미할당 자산 풀 확인 가능.

### 시나리오 E — 라이드 admin, 권한자 추가

1. `/RideAssets` 진입 (admin 권한으로 모든 탭 표시).
2. 「⚙ 권한자 관리」 탭 (admin 전용 표시).
3. 직원 검색 → 「권한자로 지정」 클릭 → `ride_asset_admins` INSERT.
4. 즉시 해당 직원의 권한 자동 부여 (재로그인 불필요 — `usePermission` hook 활용).

## 3. 페이지 구조 — NavTabs 8개

| # | 탭 | 표시 조건 | 설명 |
|---|----|---|---|
| 1 | 📋 전체 자산 | 권한자 | 모든 자산 (필터: 카테고리/상태/매칭여부) — 기본 탭 |
| 2 | 🚗 차량 | 권한자 | 카테고리 VH 자산만 |
| 3 | 🪑 사무비품 | 권한자 | 카테고리 OF 자산만 |
| 4 | 💻 IT장비 | 권한자 | 카테고리 IT 자산만 |
| 5 | 💳 법인카드 | 권한자 | 카테고리 CC 자산만 |
| 6 | 📦 공통 자산 | 권한자 | `assigned_user_id IS NULL` 자산 풀 |
| 7 | 👤 내 자산 | 모든 사용자 | 본인 매칭 자산 — 일반 사용자 진입 시 기본 |
| 8 | 🖨️ QR 인쇄 | 권한자 | 다중 선택 → 라벨지 PDF 출력 |
| (admin) | ⚙ 권한자 관리 | admin만 | `ride_asset_admins` CRUD |
| (admin) | 🏷️ 카테고리 관리 | 권한자 | `ride_asset_categories` CRUD |

**진입 분기**:
- 권한자 → 기본 탭 「📋 전체 자산」
- 일반 사용자 (본인 자산 있음) → 기본 탭 「👤 내 자산」
- 일반 사용자 (본인 자산 없음) → 안내 페이지 「매칭된 자산이 없습니다」 + 「내 자산」 빈 상태

## 5. 모바일 QR 스캔 페이지 — `/RideAssets/qr/[token]`

### 5.1 라우트
- `app/(employees)/RideAssets/qr/[token]/page.tsx` — Next.js dynamic route
- token = `ride_assets.qr_token` (UUID)

### 5.2 표시 항목 (인증 필수)

```
┌─────────────────────────────┐
│ [블루 도트 점] 라이드 자산        │
│ ───────────────────────────  │
│                              │
│  IT-2026-0001               │
│  ThinkPad X1 Carbon Gen11   │
│                              │
│  카테고리   💻 IT장비          │
│  상태      🟢 사용 중          │
│  사용자    홍길동 (본인) ★      │
│  위치      3F 개발팀          │
│  메모      배터리 교체 이력 1회 │
│                              │
│ ┌──────────────────────────┐ │
│ │ 📍 위치 업데이트            │ │
│ │ 📝 메모 추가                │ │
│ └──────────────────────────┘ │
│                              │
│  취득일 2026-01-15           │

```

### 5.3 분기 동작
- 본인 자산 → 위치/메모 편집 버튼 활성
- 타인 자산 (일반 사용자) → 자산명/카테고리만, 사용자명 「***」 마스킹
- 타인 자산 (권한자/admin) → 전체 정보
- 미할당 자산 → 「공통 자산입니다」 안내 + 위치만 표시
