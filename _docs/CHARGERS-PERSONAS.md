## 0. 어제 b-1 폐기 결정

2026-05-21 PR-6.14.b-1(`ride_chargers` 단순 자산 CRUD)은 **전면 폐기**하고 새로 설계한다.
사유: 5만기 규모 + 7단계 워크플로우(스케줄링→현장작업→보고서→검수→정산)는
"등록 버튼 + 표 한 장" 구조로 불가능. 메인 세션에 폐기 안내 완료(Rule 21).

## 1. 운영 사실 (Rule 25 — 도메인 인터뷰 결과)

| 항목 | 사실 |
|------|------|
| 사업 구조 | 라이드주식회사(MT팀)가 충전사업자 **플러그링크(PlugLink)** 의 전기차 충전기를 점검·유지보수하는 **위탁 수행** |
| 자산 규모 | 전국 최대 **5만기** (충전기 device 기준) |
| 자산 마스터 출처 | **구글시트** — 사무실직원이 신규 물량을 계속 추가 등록 (마스터 변동 잦음) |
| 식별 체계 | 권역 > 개소명(충전소) > 주소 > 상세위치. 공식 충전기 ID는 **일부만** 보유 |
| 점검 유형 | 정기점검(월간 / 분기 / 반기), 시공감리(완료 기한 **10일 이내**), **고장점검**(신고 기반), 현장전환 — 총 6종 |
| 충전사업자 변동 | 플러그링크(PlugLink) → **한화모티브** 인수 → "현장전환" 캠페인 발생 배경 |
| 마스터 현황 | 26년 상반기 — 감리 5,141 / 정기점검 반기 7,176·분기 62·월 976 / 현장전환 6,500기 |
| 충전기 타입 | 완속 / 급속 (보고서 양식·정산 단가 분기 기준) |
| 점검 단위 | 보고서 1건 = **충전소(개소) 1곳** 점검. 한 충전소에 충전기 여러 기 |
| 보고서 양식 | 점검 유형별 양식 상이 — 충전사업자 지정. 실파일: 작업폴더 외부 `충전기점검보고서/` |
| 정산 | 점검 유형 × 충전기 타입별 **단가표** + 별건(수동 입력). 작업 완료 후 충전사업자에 정산 요청 |
| 사업자 제출 | 보고서를 충전사업자 구글드라이브에 **수동 업로드**. ERP는 "업로드함" 상태만 추적 (자동 연동 X) |
| 검수 | 충전사업자 측 검수 → "완료" 또는 "사진 보완 요청" → 재작업 (실파일 접두어로 확인) |

## 2. 페르소나

### P1. 현장직원 (점검원)
- 점검 **일정 등록(스케줄링)** 을 직접 수행
- 전체 점검 리스트를 보되 **본인 담당 건만 필터링** 가능 (스케줄 관리용)
- 현장 출동 → 점검 작업 수행 → **보고서 기준에 맞춰 사진 촬영·업로드**
- 점검 유형별 양식에 맞춰 **보고서 작성**
- 권한: 점검 일정·작업·사진·보고서 (정산 입력 불가)

### P2. 사무실직원
- 작성된 **보고서 검수** → 통과 / 재작업 요청
- **구글시트 관리** + 신규 충전기 **추가 등록** (마스터 유지보수)
- 충전사업자 구글드라이브 업로드 후 ERP에 "업로드함" 체크
- 권한: 마스터 등록·수정, 보고서 검수 (정산 확정 불가)

### P3. 관리자
- **정산 입력·확정** (전용 권한 — 다른 페르소나 불가)
- 전체 현황 모니터링 (권역별 진행률 / 미정산 / 기한 임박)
- 권한: 전 기능 + 정산

## 3. 7단계 운영 시나리오

| # | 단계 | 주체 | 화면/동작 | 데이터 |
|---|------|------|----------|--------|
| 1 | **마스터 적재** | 사무실직원 | 구글시트 → ERP 임포트, 신규 추가 등록 | 충전소·충전기 마스터 |
| 2 | **스케줄링** | 현장직원 | 점검 유형별(월/분기/반기/감리) 일정 생성 + 담당자 배정 | 점검 일정 |
| 3 | **현장 작업** | 현장직원 | 출동 → 점검 수행 → 보고서 기준 사진 촬영·업로드 | 점검 결과 + 사진 |
| 4 | **보고서 작성** | 현장직원 | 유형별 지정 양식에 데이터 채워 보고서 생성 → 다운로드 | 보고서 파일 |
| 5 | **사업자 제출** | 사무실직원 | 충전사업자 구글드라이브에 수동 업로드 → "업로드함" 체크 | 제출 상태 |
| 6 | **검수** | 사무실직원 / 사업자 | 검수 → 완료 또는 사진 보완 요청 → 재작업 | 검수 상태 |
| 7 | **정산** | 관리자 | 완료 건 단가표 적용 정산 생성 → 확정 → 사업자 정산 요청 | 정산 |

> 상태 흐름: `예정 → 작업중 → 보고서작성 → 제출 → 검수중 → (재작업 ↺ / 완료) → 정산완료`
