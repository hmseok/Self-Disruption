================================================================================
AUTHENTICATION & SECURITY
================================================================================

Pattern Implemented in All New Routes:

1. JWT Token Extraction
   - Reads Authorization header
   - Expects "Bearer <token>" format
   - Decodes JWT payload manually (no external library needed)
   - Extracts user ID from payload.sub or payload.user_id
   
2. User Verification
   - Queries profiles table to get user's company_id and role
   - Returns 401 Unauthorized if token invalid or user not found
   - All queries filtered by user.company_id for isolation
   
3. Data Serialization
   - Converts BigInt to string to avoid JSON serialization issues
   - Uses standard JSON.parse/stringify with custom replacer
   - Works with Prisma's raw SQL queries

Frontend Auth Pattern:

```typescript
async function getAuthHeader(): Promise<Record<string, string>> {
  const { data: { session } } = await supabase.auth.getSession()
  const token = session?.access_token
  return token ? { Authorization: `Bearer ${token}` } : {}
}

// Usage in every fetch call:
const headers = {
  'Content-Type': 'application/json',
  ...(await getAuthHeader())
}
```

ERROR HANDLING
================================================================================

Frontend Error Handling Pattern:

1. Try-Catch Blocks
   - All async fetch() calls wrapped in try-catch
   - Console logging for debugging
   - User alerts for critical failures
   
2. Response Validation
   - Check !res.ok before processing
   - Parse json.error message
   - Provide meaningful error messages to users
   
3. State Management
   - Fallback to empty arrays on error
   - Loading states properly managed
   - No silent failures

Backend Error Handling Pattern:

1. Authentication Failures
   - Return 401 with message: '인증 필요'
   
2. Validation Errors
   - Return 400 with descriptive message
   
3. Server Errors
   - Return 500 with error.message
   - Log to console for debugging
