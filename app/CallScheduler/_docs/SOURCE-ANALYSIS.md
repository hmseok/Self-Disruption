# CallScheduler — 원천 자료 분석 (5월 스케줄)

> 입력 자료: 사용자 제공 5월 근무 스케줄표 스크린샷 (2026-05-03 수신)
> 이 문서는 데이터 모델·UI 설계의 1차 기준이 됩니다. 향후 양식 변경 시 본 문서를 먼저 갱신.

## 2. 근무자 명단 (식별된 16명)

**주간 그룹 (12명)**: 박지훈, 이혜경, 김현정, 정지은, 추경희, 정우진, 서민아, 이경미, 전소현, 안경희, 박혜정, 유수정
**야간/저녁 그룹 (4명)**: 정동민, 전정연, 윤민진, 전유하

> 동일 인원이 다중 슬롯 점유 가능 (예: 안경희 → 주말 L07, 평일 L09 일부)

## 3. 특수 코드 (Special Code) — 파란색 굵은 텍스트

| 코드 | 의미 | 처리 |
|------|------|------|
| `오전F` | 오전 자유(휴무) | 슬롯 점유, 근무자 NULL, 색상=파란틴트 |
| `오후F` | 오후 자유(휴무) | 동일 |
| `오전반차` | 오전 반차 | 슬롯 점유, 근무자=배정자, 코드만 추가 |
| `오후반차` | 오후 반차 | 동일 |
| (회색 음영) | 휴무/공석 | 슬롯 비배정 (NULL row 또는 row 생략) |

## 4. 셀 컬러 토큰 (야간/저녁조 사람별)

엑셀 원본에서 사람별 고정 배경색 → 시각 식별 보조

| 근무자 | hex (추정) | 디자인 토큰 매핑 |
|------|------|------|
| 정동민 | #DBEAFE (연파랑) | `COLORS.bgBlue` |
| 전정연 | #F3F4F6 (회색) | `COLORS.bgGray` |
| 윤민진 | #DCFCE7 (연녹색) | `COLORS.bgGreen` |
| 전유하 | #FEF3C7 (연황색) | `COLORS.bgAmber` |

> 본 톤은 worker.color_tone 컬럼에 enum(`blue|gray|green|amber|violet|red|none`) 저장. 신규 인원은 입력 시 선택.

## 5. 캘린더 구조

- 그리드: 7열(일~토) × 6주
- 행: 시프트 슬롯 13개 + 헤더 1행
- 셀 = 1일 1슬롯 = 1배정(또는 NULL)
- 5월 1일 = 금요일 (2026-05-01) → 월간 첫 행은 1~2일만, 마지막 행은 31일 단독
- 주말 음영 처리 (일/토 헤더 톤 변경)

## 6. 분석 KPI 후보 (목록 페이지 상단 KPI Strip)

1. **인당 평균 근무시간** (월 합산 ÷ 근무자 수) — 균형도 지표
2. **시프트 충원율** (배정 슬롯 / 정의 슬롯) — 빈 칸 비율 역지표
3. **야간 로테이션 편차** (야간조 4인 야간 일수 표준편차)
4. **반차/F 사용 합계** (인당)
5. **주말 근무 균형** (인당 토/일 근무 일수)

## 7. "체크 배포" 의미 정의 (사용자 추가 확인 필요 항목)

스크린샷만으로는 정확한 정의가 불명. 합리적 가정 2가지:

- **(A) 작성된 스케줄을 근무자에게 배포(공지)** — 잔디/이메일/공유링크. **본 MVP 1차 가설**
- **(B) 근무자가 출근 시 본인 시프트 체크인/체크아웃 기록**

→ MVP는 **(A) 가설**로 설계. (B) 요구되면 `attendance_logs` 테이블 추가로 확장 가능 (스키마는 호환).
