# CallScheduler — 운영 사실 (Domain Operations)

> CLAUDE.md 규칙 25 의무 — 데이터 모델 / 디폴트 값 / 자동 로직의 근거 문서.
> 사용자 인터뷰 후 사실관계를 기록 (운영 패턴 변경 시 즉시 갱신).

## 1. 운영 시간

| 항목 | 사실 |
|------|------|
| 운영 시간 | **24/365** (24시간 365일 무중단) |
| 공휴일 | **누군가는 항상 일함** — 회사 차원 일괄 휴무 없음 |
| 야간 시프트 | L13 (20:30~08:30) — 익일 종료, 12시간 |
| 저녁 시프트 | L12 (19:00~23:00) — 단시간 |

**데이터 모델 영향**:
- `cs_holidays.exclude_auto` 디폴트 = `false` (자동 제외 X)
- 자동 생성 API `skip_holidays` 옵션은 default false
- 공휴일 자체는 회사가 쉬는 날이 아닌 **참고용 마커** + 직원 휴가 적용 도구

## 3-B. 자동 생성 알고리즘 v3 (PR-2QQ-d-3)

**옵션** (AutoGenerateDialog):
- `use_priority` (기본 ON) — priority + dow_avoid + cycle/dow_only 패턴 + 균등 분배
- `enforce_min_coverage` (기본 ON) — cs_group_min_coverage 의 요일별 최소 인원 충족
- `skip_holidays` (기본 OFF — 24/365)
- `mark_leaves` (기본 ON)
- `overwrite_existing` / `clear_first` (manual_lock 셀 항상 보존)

**가중치 정렬 우선순위**:
1. `priority_level` ASC (P1 우선)
2. `preferred_dow_avoid` 포함 시 후순위
3. `required_days_per_month` 미달 워커 우선
4. `by_dow[date.dow]` ASC (해당 요일 적게 한 사람 우선) ← 금/일 균등
5. `total` ASC (전체 적게 한 사람 우선)
6. 마지막 배정일과 거리 DESC (오래 안 한 사람 우선)

**필터**:
- locked (manual_lock=1) 워커는 후보 제외 (이미 박힘)
- 휴가 종일 (`special_code = 'off'`) 워커 제외
- `max_days_per_month` 초과 워커 제외
- `cycle_days_on/off/start` 정의 시 휴무 phase 워커 제외
- `preferred_dow_only` 정의 시 한정 요일 외 워커 제외

**부족 경고**:
- 그룹 멤버 < 그날 min_workers 시 `summary.warnings` 에 `{group, date, missing}` 누적

**예시 — 야간 그룹 시뮬레이션**:
```
정동민: P1, ext, cycle_on=2, cycle_off=2, start=2026-05-01
전정연: P2, dow_avoid=일·금
윤민진: P2, dow_avoid=일·금
야간콜 group min: 매일 2 + 금 3 + 일 1

5/1 (금) min=3 → [정동민(P1), 전정연(P2,금회피), 윤민진(P2,금회피)]
5/3 (일) min=1 → 정동민 cycle 휴무 → [윤민진] (전·윤 일회피 동률, last_date 거리)
5/4 (월) min=2 → 정동민 cycle 휴무 → [전정연, 윤민진]
5/5 (화) min=2 → 정동민 근무 재개 → [정동민, 전정연 또는 윤민진(동률)]
```

## 3-A. 그룹 최소 인원 (PR-2QQ-d-2)

각 그룹은 매일 디폴트 인원 + 요일별 예외 설정 가능 (cs_group_min_coverage):

| 예시 그룹 | 디폴트 | 금 | 일 | 의미 |
|---------|--------|-----|-----|-----|
| 야간콜 (L13) | 2명 | 3명 (피크) | 1명 | 정동민 + 우리 직원 1명 기본, 금 +1, 일 정동민만 |
| 주간 09-18 (L05) | 1명 | - | - | 매일 1명 — 평일만 패턴 |
| 주말 보강 (L02) | - | - | - | 주말만 패턴 사용 → 디폴트 의미 X |

**자동 생성 알고리즘** (PR-2QQ-d-3 예정):
1. 슬롯 × 날짜 → 그룹 → 그룹 멤버 풀
2. (그룹, 요일) 의 min_workers 만큼 워커 선택
3. 워커 가중치 정렬 (priority + dow_avoid + required + 균등)
4. manual_lock 셀은 보존

**graceful**: cs_group_min_coverage 미적용 시 min=1 디폴트 (이전 동작과 동일)

## 4. 휴가 발급 + 사용 정책 (Ride Inc. 콜센터)

### 4-A. 발급 단위

| 종류 | 발급 단위 | 기본 일수 | quota 추적 |
|------|---------|---------|-----------|
| **연차** (annual) | 연 1회 | 15일 | ✅ |
| **패밀리데이** (familyday) | 월 1회 | 1일 (12개월×1) | ✅ |
| **병가** (sick) | 연 단위 (선택) | 5일 | ⚠ 선택 |
| **공휴일 휴무** (holiday) | 직원별 시프트 셋팅 | 무발급 | ❌ |
| **무급** (unpaid) | 이벤트별 | 무발급 | ❌ |
| **경조** (family) | 이벤트별 | 무발급 | ❌ |
| **기타** (other) | 이벤트별 | 무발급 | ❌ |

### 4-B. 사용 단위 (시간 — 회사 정책)

| 종류 | 시간 단위 default | 사용 시 차감 |
|------|----------------|-----------|
| **연차 종일** | full (8h) | 1일 |
| **연차 반차** | am / pm (각 4h) | 0.5일 |
| **연차 단축** | custom Nh | N/8 일 |
| **패밀리데이** | **custom 3시간 일찍 퇴근** | 3/8 = 0.375일 |
| **반차 (별칭)** | am 또는 pm 4h | 0.5일 |
| **병가/무급/경조/기타** | full (사용자 변경 가능) | 종일=1일 |

**전년 이월** (carried_over_days) — 옵션, 기본 0
**시간 환산**: am_pm='custom' 일 때 hours / 8 = 일 차감

### 4-C. 자동 default 적용 (UI)

설정 → 직원 휴가 → [+ 등록] / [편집] 시 종류 클릭하면 회사 정책 default 자동 적용:
- 연차 → full (8h)
- 패밀리데이 → custom 3h
- 그 외 → full (사용자 변경 가능)

**빠른 프리셋 버튼** (폼 안):
- ⏰ 반차 (오전 4h) / ⏰ 반차 (오후 4h)
- 🏃 패밀리데이 (3h)
- ☀ 종일 (8h)

코드 위치: `app/CallScheduler/utils/leaveDefaults.ts`
**다른 회사 정책으로 변경 시** 이 파일 수정 (또는 추후 cs_leave_types 마스터로 분리).

## 9. 야간 슬롯 안전 가드 (2026-05-05 사용자 인터뷰)

PR-2SS 시리즈 도입의 운영 사실 근거:

| 항목 | 사실 | 데이터 모델 디폴트 |
|------|-----|-----------------|
| 야간 가산율 | **없음** | `night_premium_rate = 0` (PR-2SS-e 컬럼 추가, 가산 안 함) |
| 연속 야간 한도 | 회사 규정 X / **운영 기본 3일** | `slot.max_consecutive_days = 3` (overnight 만, PR-2SS-c 활용) |
| 야간 후 휴식 | 규정 X / **자연 16시간** | `slot.next_day_blocking_hours = 16` (overnight 만, PR-2SS-b 활용) |
| 휴일 야간 특수 | 평일과 동일 | 별도 컬럼 X (PR-2SS-e 도입 보류) |
| 신입 야간 | **금지 (운영 정책)** | `slot.min_seniority_months = 6` (overnight 만, PR-2SS-d 활용) |

**알고리즘 효과**:
- PR-2SS-b: 야간 끝나고 16시간 안에 다른 슬롯 시작 못함 → 야간조가 다른 그룹에도 멤버여도 안전
- PR-2SS-b: 같은 워커 같은 날 두 슬롯 시간 겹침 자동 detect → manual_lock 박을 때 사전 경고

**5월 운영 케이스 회귀 검증**:
- 야간조 (정동민·전정연·윤민진) 모두 cs_shift_groups 의 야간콜 그룹 단독 멤버
- 다른 주간 그룹 멤버 X → 자연 격리되어 익일 휴식 가드 발동 안 함 (회귀 0)
- 매니저가 정동민을 manual_lock 으로 주간 슬롯에 박을 경우만 가드 발동
