# CallScheduler v2 — 재구성 설계서

> 트리거: 사용자 피드백 (2026-05-03)
> "셋팅 ui랑 표출 ui랑 휴가반 화면 사이즈도 그렇고 좀 직원별 시간이나 스케줄 같은 것도 필요할것같고
>  스케줄 매니저입장에서 직접 짜고 배포한다고 생각하고 작성자 입장에서 또 직원입장에서 고려하여 조금 구성좀해줘"

## 0. v1 한계 (1차 MVP 회고)

| 문제 | 원인 | 영향 |
|------|------|------|
| 한 페이지에 작성·조회·분석·배포 다 우겨넣음 | 페르소나 미분리 | 매니저든 직원이든 같은 화면 — 직원에게 과한 기능 |
| 마스터(시프트·워커) 관리 화면 없음 | 시드만 있고 UI 없음 | 신규 워커 추가, 시프트 변경 불가능 (DB 직접 수정해야) |
| 휴가/반차가 셀 단위 special_code 만 | 일자 범위 휴가 없음 | "5/10~5/15 휴가" 같은 일괄 처리 불가, 신청/승인 흐름 없음 |
| 캘린더 폭 활용 부족 (max-width 1600) | 화면 안에 분석 패널까지 | 31일×13행 한 화면에 다 안 들어옴 |
| 직원 본인 일정 보기 없음 | 매니저 시점만 | 직원이 본인 스케줄 빠르게 못 봄 |

## 2. v2 페이지 맵

```
/CallScheduler                  ← (M) 메인 — 월간 캘린더 작성/배포 (풀폭)

/CallScheduler/[id]             ← (M) 특정 월 상세 (현재 [id] → 라우트 유지, 레이아웃 풀폭화)
/CallScheduler/me               ← (E) 직원 마이 페이지 — 본인만 표출
/CallScheduler/employees        ← (M) 워커 KPI 비교 (분석 전용 페이지)
/CallScheduler/employees/[wId]  ← (M+E) 한 명 상세 (월간 본인 시간표 + 누적)
/CallScheduler/leaves           ← (M+E) 휴가 신청/승인 — 매니저 모드 / 직원 모드 자동 전환

권한 가드: `verifyUser` + 페이지별 role 체크 (Phase 2 단순 구현 — 모든 인증자 통과, role 분기는 UI 측에서)

## 3. 레이아웃 개선

### 3.1 메인 캘린더 (`/CallScheduler/[id]`)

**v1 (현재)**:
- max-width 1600px → 캘린더 + 우측 분석 360px = 캘린더 좁음

**v2**:
- max-width 제거 → 화면 폭 100% 활용
- 분석 패널은 우측 **토글 드로어** (기본 닫힘) — 햄버거 같은 [📊 분석] 버튼
- 셀 폭 60→44px, 높이 28→24px → 31일×13행 한 화면 내 (1440px+)
- 좌측 시프트 라벨 sticky 유지
- 모바일 (768 미만): 가로 스크롤 + 셀 크기 그대로

### 3.2 액션 바 재배치

```
┌─────────────────────────────────────────────────────────────────────┐
│ ← 목록 │ 2026년 5월 │ [초안] │ ... │ [📊분석] [⚡배포] [⋯더보기]   │

- 더보기(⋯) 메뉴: 공지됨/초안 토글, 복제, 엑셀 내보내기, 삭제

## 4. 휴가/반차 관리 — `cs_leaves` 신설

### 4.1 데이터 모델

`cs_leaves` (범위 기반 휴가)
- id, worker_id, leave_type (annual / sick / unpaid / etc), start_date, end_date
- am_pm ('full' | 'am' | 'pm') — 반차일 경우
- status ('requested' | 'approved' | 'rejected' | 'canceled')
- requested_by, requested_at, approved_by, approved_at, reason

### 4.2 처리 흐름

```
직원 → 휴가 신청 (cs_leaves INSERT, status=requested)
   ↓
매니저 → 승인 (status=approved, approved_at=now)
   ↓
시스템 → 해당 범위의 cs_assignments 자동 special_code 반영
   · full → 'off'
   · am   → 'am_half' (오전 시프트면) / 'am_free'
   · pm   → 'pm_half' / 'pm_free'
   ↓
KPI/캘린더 즉시 갱신
```

### 4.3 v1 special_code 와의 관계

- v1: cs_assignments.special_code 직접 셀에 저장 (즉시 반영)
- v2: 일회성·반복은 v1 그대로, **승인 흐름이 필요하면 cs_leaves 거침**
- 둘 다 결국 cs_assignments 에 표출됨 → 캘린더는 변경 없음

## 6. 워커별 분석 페이지 (`/CallScheduler/employees`)

매니저 전용. 한 달의 모든 워커를 가로 비교:

```
이름      그룹  시프트  시간  야간  반차  F   균형도
박지훈    주간   21    168h   0    1    0    +5%
이혜경    주간   20    160h   0    0    1    +1%
...
정동민    야간    8     96h   8    0    0    -8%
```

- 균형도 = (인당시간 - 평균) / 평균
- 색상 ±15% 초과 시 amber, ±30% 초과 시 red
- 클릭 → `/CallScheduler/employees/[wId]` 한 명 상세

## 8. 결정 사항 (사용자 확정 — 2026-05-03)

### 8.1 휴가 흐름 = **(B) 매니저 직접 입력 단일 단계**

- `cs_leaves` 신청/승인 테이블 **신설하지 않음**
- 매니저가 캘린더 셀에서 special_code (`off`/`am_half`/`pm_half`/`am_free`/`pm_free`) 직접 클릭 (현재 v1 동작 그대로)
- **PR-2D 휴가 별도 페이지 폐기**, 대신 **일괄 입력 도구만 추가** (예: "5/10~5/15, 박지훈, 휴무" 한 번에)
- v1 special_code UI 강화로 충분

### 8.2 직원 진입 = **영구 토큰 링크**

직원당 1회 발급, 평생 북마크 가능. 매월 새 공유 X.

#### 데이터
- `ride_employees.public_token CHAR(32) UNIQUE NULLABLE` 컬럼 추가
- 매니저가 settings에서 [발급] 클릭 시 crypto.randomBytes(16).toString('hex') 생성
- [재발급]으로 토큰 invalidate + 새로 생성 (이전 링크 만료)

#### URL 구조
- `/CallScheduler/e/[token]` — 비로그인 영구 직원 링크
  - 토큰으로 ride_employee 조회 → cs_workers 의 employee_id 매핑 → 본인 cs_assignments 만 표출
  - **status='published' 스케줄만 노출** (초안 상태는 안 보임)
  - 라우트는 `verifyUser` 우회 + 토큰 검증으로 대체

#### 비교: `/CallScheduler/me`
- 로그인 필요 (profiles 인증)
- 본인 + 매니저 권한 시 다른 워커 선택 가능 (관리자 미리보기)
- 토큰 링크와 같은 본문 컴포넌트 재사용
